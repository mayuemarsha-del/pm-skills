# PRD Writing Checklist

## Before writing

- [ ] 先确认任务目标
- [ ] 查相关系统现状和最近结论
- [ ] 明确对象粒度
- [ ] 明确入口位置
- [ ] 明确是否复用现有页面框架
- [ ] 明确本期做 / 不做

## While writing

- [ ] 背景只写问题，不提前展开解决方案
- [ ] 先写产品结构和页面定位
- [ ] 字段约束收敛在字段表里
- [ ] 规则按类型归类
- [ ] 交互、边界、异常处理有写
- [ ] 同类模块口径统一

## Before delivery

- [ ] 与系统现状不冲突
- [ ] 看起来像正式 PRD
- [ ] 没混入过多技术实现
- [ ] 如果有原型，已检查原型和正文一致
- [ ] 用户指出过的问题已联动复查
- [ ] 如果是 Agent / 算法相关需求，已补充后端与算法交互流程
- [ ] 如果是 Agent / 算法相关需求，已补充算法处理流程
