# AgentBuilder 领域知识

写涉及产品体系、模块定位、Agent 链路或迁移策略的 PRD 时按需查阅。一般写法规则的小任务（重写背景段、整理某节、改字段表）不必加载本文。

## 产品体系

1. **AgentBuilder**：新一代客服 Agent 平台（工作台 / 数据中心 / 评测中心 / Agent 管理 / 资源中心）。
2. **核心概念**：Agent、知识库、Skill、Tool（FC / MCP）、Flow。
3. **用户角色**：产品经理 / 实施人员、运营人员、客服主管、管理员、技术支持。

## 核心模块定位

| 模块 | 定位 | 用户频率 |
|------|------|----------|
| 工作台 | AI 对话操作入口，驱动 Agent 构建 / 诊断 / 优化 | 最高频 |
| Agent 管理 | Agent 全生命周期配置 + 接待记录 | 高频 |
| 资源中心 | 知识库 / Skill / Tool 统一管理 | 中频 |
| 数据中心 | 全局数据观测与问题发现 | 中频 |
| 评测中心 | 系统化度量 Agent 效果 | 低频 |

## 问答链路关键概念

1. **Skill**：专项处理单元，含提示词 + 内部调用逻辑；命中时加载提示词，完成后去除。
2. **Tool（FC / MCP）**：外部能力接口；失败时按降级策略处理，不向用户暴露错误。
3. **知识召回**：向量 + 关键词混合检索；QA 类优先精确匹配。
4. **多路冲突**：Tool 实时结果优先于知识库静态内容。
5. **转人工**：AI 判断为主 + 硬规则兜底（重复 / 负向情绪 / 连续无法回答）。
